# OmniUI Implementation Details

## 1. Hybrid Application Architecture
The `OmniApplication` class serves as the unified entry point for the BobUI OS toolkit. It extends `QApplication` to maintain 100% Qt 6 parity while introducing specialized bootstrap logic for secondary frameworks.

### 1.1 JUCE Integration Bridge
The `initializeJuce()` method prepares the multimedia subsystem. By bypassing the default JUCE modal message loop and instead hooking into the Qt `QAbstractEventDispatcher`, we allow high-performance audio/visual components to run within standard Qt windows without context switching or event starvation.

### 1.2 JuceWidget Wrapper
The `JuceWidget` class is a high-performance bridge that hosts a `juce::Component` inside a `QWidget`.
- **Rendering**: Uses an opaque paint event strategy. Future versions will bridge JUCE's low-level OpenGL context directly into the Qt RHI (Vulkan/DirectX) via shared texture handles.
- **Event Routing**: Mouse and keyboard events are captured by the Qt widget and translated into JUCE coordinate spaces and event types.

## 2. Multi-Input & Multi-Cursor System
The `OmniInputManager` is a singleton responsible for managing the identities of all connected HID devices.

### 2.1 Device Identity
Unlike standard OS paradigms that merge all mice into one system cursor, BobUI treats every device as a unique entity. This is the foundation for:
- **Local Multiplayer**: Multiple users interacting with the same screen.
- **Collaborative Workspaces**: Designers and developers having independent focus points.

### 2.2 Input Event Tagging
The next phase of implementation involves modifying `QGuiApplication` to ensure every event generated by the platform plugins carries the `DeviceId` from `OmniInputManager`.

## 3. TypeScript / QML Transpilation Path
The TypeScript definitions in `OmniUI/ts/definitions/omni.d.ts` provide the developer-facing API.
- **Omni.Application**: Exposes the `OmniInputManager` state to the frontend.
- **Omni.JuceComponent**: Allows frontend developers to instantiate and configure complex JUCE nodes as if they were standard HTML/QML elements.
